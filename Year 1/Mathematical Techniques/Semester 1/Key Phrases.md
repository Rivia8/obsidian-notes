# Phrases

## Key Phrases:
This includes phrases such as: 'and', 'or', 'implies' ('if' and 'only if'), 'there exists', and 'for all'.
When looking at formal logic we see that these phrases correspond to logical operations also known as connectives and in the derivation system we study at that point the principles from this section are formalised. 

**And**:
We use this word to connect several statements or properties - all of them must hold.
![[Pasted image 20241005140302.png#invert]]

**Or**:
We connect two statements or properties with 'or' if at least one,  (but possibly both of them) hold. 
![[Pasted image 20241005141020.png#invert]]

**Implies**:
This phrase tells us that if the first statement holds then so does the second. 
![[Pasted image 20241005164516.png#invert]]

**If and only if**:
This phrase is merely a short-cut. When we say that Statement 1 (holds) if and only if Statement 2 (holds) then we mean by this that both, Statement 1 implies Statement 2 and Statement 2 implies Statement 1. 
![[Pasted image 20241005164737.png#invert]]

**For all:**
![[Pasted image 20241005164838.png#invert]]
A 'for all' statement should have two parts:
- For which elements that are making the claim? There should be a set associated with this part of the statement.
- What property or properties do these elements have to satisfy? There should be a statement which specifies this. 

**There exists:**
![[Pasted image 20241005171402.png#invert]]
Whenever a statement is made about existence there should be two parts to it: 
• Where does the element exist? There should always be a set associated with the statement. In the above example, 𝑘 had to be an element of N (and indeed the existence of some 𝑟 ∈ R with the same property would completely change the definition and make it trivial). 
• What are the properties that this element satisfies? There should always be a statement which specifies this. In the example above the property is 𝑚 = 𝑘𝑛 (for the given 𝑚 and 𝑛).
