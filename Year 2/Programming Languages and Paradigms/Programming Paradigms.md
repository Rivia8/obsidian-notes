# Programming Paradigms

A programming paradigm is a **style of programming**. The style defines how the programmer describes a program within its source code:
- Data that it uses,
- Computations manipulating the data,
Paradigms are high level categories used to classify programming languages.

## The Relation to Programming Languages

All programming languages fall within at least one paradigm. 
This makes it easy to program in the style defined by the paradigm in question.

*For example: a lot of operations look pretty similar in C++ and C# because both languages fall within the Object-Orientated Programming Paradigm.*

A lot of languages can be classified within several paradigms.

![[paradigms.svg#invert]]

Given a particular software engineering problem, certain paradigms are better suited than others.


## A Programming Language is a Programming Style

The paradigm characterises how the programmer **defines** computation, it is generally a sequence of statement performing various operations, in various order - for example sequentially or in parallel.
- The computations can be decomposed into units named functions.

## Picking a Programming Paradigm

Because some paradigms are more efficient than others to solve a given, problem when one needs to develop software, choosing a language/paradigm has huge consequences on the efficiency and clarity.
