# Physical Data Model


When it comes to database creation we start with a conceptual model, then move to a Logical Data Model in the form of an [[EER Modelling|ER Diagram]] and then we produce a **Physical Data Model:**

![[Pasted image 20251023213540.png#invert|800]]



Usually once we have a normalised ER Diagram we add things to it to form the PDM:
1. Naming conventions,
2. Reserved keywords
3. Data types
4. Constraints
