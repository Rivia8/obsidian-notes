# Informed Search Strategies

This is unlike [[Uninformed Search Strategies|uninformed search strategies]] where we use domain specific hints about the location of goals. With this we can find solutions more efficiently than an uninformed strategy. 
The hints come in the form of a **heuristic function** denoted as $h(n)$:
$$h(n) = \mbox{estimated cost of the cheapest path from the state at node $n$ to a goal state}$$

So an example of this would be if we are looking for optimal routes, we can estimate the distance from the current state to a goal by computing the straight-line distance on the map between the two points.

## Greedy best-first Search

This is a form of [[Search Algorithms#Best-First Search|best first search]] that expands the first node with the lowest $h(n)$ value.

Greedy best-first graph search is complete in finite spaces, but not in infinite ones. The worst-case time and space complexity is $O(|V|)$. 
With a good heuristic function the complexity can be reduced substantially to $O(bm)$.

## A* Search

The most common informed search algorithm is **A* search**, a best-first search that uses the evaluation function: $$f(n) = g(n) + h(n)$$
- $g(n)$ = path cost from the initial state to node $n$,
- $h(n)$ = is the estimated cost of the shorted path from $n$ to a goal state,

this gives us:
- $f(n)$ = estimated cost of the best path that continues from $n$ to a goal state.

A* search is complete. Whether A* is cost-optimal depends on the **heuristic** and its properties:
A key property is if its **admissible** which is one that never overestimates the cost to reach a goal - this means that an **admissible** heuristic is **optimistic**.

A slightly stronger property is called **consistency**. A heuristic $h(n)$ is consistent if, for every node $n$ and every successor $n'$ of $n$ generated by an action $a$ we have:
$$h(n) \le c(n,a,n') ~ + ~ h(n') $$
This is a form of the **triangle inequality**, which is that a side of a triangle cannot be longer than the sum of the of the other two sides:

![[Pasted image 20251003081803.png#invert]]

Every consistent heuristic is admissible (but not vice versa), so with a consistent heuristic, A* is cost-optimal. In addition, with a consistent heuristic, the first time we reach a state, it will be on an optimal path, so we never have to re-add a state to the frontier, and never have to change an entry in reached, but with inconsistent heuristics, we may end up with multiple paths reaching the same state, and if each new path has a lower path cost than the previous one, then we will end up with multiple nodes for that state in the frontier, costing both time and space.
