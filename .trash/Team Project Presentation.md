## Design

The first thing you see when you open the website *is* the dashboard
When it came to producing the dashboard: 

- We used cards to link to each page and function of the site
- They have simple icons, that are also present throughout other sections of the website.

Now onto the next part of the design:

- One of the key elements of the design is the sidebar, it acts as the primary navigation tool for the user to access the core functions once you're within a page not at the dashboard.
- It retains the colour scheme of the overall website and is easy and intuitive to use.
- To make it look better and more appealing there is a sleek hover animation (that you saw during the live demo) which was implemented through CSS styling techniques.

## Organisation

When it came to the tools for communication:

- Main platform was WhatsApp - where we had quick updates and reminders 
- We also had online meetings for weekly sync-ups for task planning and progress reviews.

For collaboration and progress tracking:

- We mainly used Git commits to view changes and backlogs of any updates made to the code
- We also used code comments for inline communication if someone else's code was maybe too hard to interpret.

For the roles we simply split and assigned it based on the team's own strength's and weaknesses. The roles included: Front-end dev, Back-end dev and Data Structures lead.
